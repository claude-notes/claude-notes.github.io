Anthropic突如其来的限额！Claude Code还能继续放心撸代码吗？

就在今天，Anthropic突然为自家代码助手 Claude Code 开出“限速令”，在X上发布称将从 2025 年8月28日起将对 Claude Pro 和 Max 套餐实行每周使用上限，并沿用此前每 5 小时刷新的会话上限。

新规到底限了什么？

新政策将根据订阅档位按小时计量。

根据Anthropic官方公告，新的每周使用限额将主要影响Pro计划（每月20美元）以及Max计划(每月100美元和200美元)的用户，预计受影响用户占比不到5%。

  • Pro 套餐（每月 20 美元）每周可用 Claude Sonnet 4 模型约 40–80 小时；

  • Max 计划

    • 100 美元档每周可用 Sonnet 4 约 140–280 小时、Opus 4 约 15–35 小时；
    • 200 美元档则可用 Sonnet 4 约 240–480 小时、Opus 4 约 24–40 小时。

假如你用量超标，官方提供按 API 标准费率购买额外配额的选项。

为何要“限速”？

Anthropic 在邮件和社区帖子中解释，Claude Code 自推出以来需求激增，有用户 24 小时挂着不下线，还有账号共享甚至倒卖的行为，拖慢了平台的整体效率。

每周限额也是为了遏制极端用法，让大多数人都能稳定使用。

开发者为啥不满？

虽然官方宣称影响的只是少数“重度用户”，但国内外很多开发者吐槽这次调整不透明。

但其实早在 7 月中旬，Anthropic 已偷偷缩紧过限制，一些 Max 用户刚打开就收到“已达到限额”的提示，却不知道自己用了多少。

更糟糕的是，Anthropic以小时而非传统 token 计费，用户无法实时查看剩余额度。一些用户抱怨 200 美元的计划比 100 美元档多花 100 美元，却只多了 5 小时的 Opus 4 时长。

社交媒体上甚至有人调侃，这是“把油枪交给加油站，司机却看不到油表”，结果大家不是省着用就是提前囤积额度。
