Claude Code 限额令自8月28日落地，开发者该怎么办呢？教你怎么使用才能玩得更久

一转眼就 8 月 28 日，新规正式上路：

  1. 保留现有每 5 小时重置一次的使用上限（老规矩不变）。

  2. 新增：每7天重置一次的总体周使用上限。

  3. 新增：Claude Opus 4 专属周使用上限（每 7 天重置一次）。

官方之前的口径说是 影响 <5% ，但对重度并发、跨仓深推这类用法，限额估摸着会更“贴脸”。

Claude Code现在到底能用多久？

注意：下面的“小时”是估算区间，实际按token触发限额，只是用小时更直观。

  • Pro（$20）：
    • Sonnet 4 每周约 40–80 小时；
    • Pro 无法在 Claude Code 使用 Opus 4。
  • Max 5×（$100）：
    • Sonnet 4 每周 140–280 小时；
    • Opus 4 每周 15–35 小时。
  • Max 20×（$200）：
    • Sonnet 4 每周 240–480 小时；
    • Opus 4 每周 24–40 小时。

官方参考文档：https://support.anthropic.com/en/articles/11145838-using-claude-code-with-your-pro-or-max-plan

把周配额 均摊到 7 天 只为便于理解（非硬性日配额）：

  • Pro ≈ 6–11 小时/天；
  • Max $100 ≈ 20–40 小时/天；
  • Max $200 ≈ 34–68 小时/天；
  • Opus 4 ≈ 2–5 小时/天。

现实中仍受5 小时窗口与7 天滚动双重约束。

用不够怎么办？

到顶后，你可以在终端选择用 API 额度“续杯”（按标准 API 费率计费），或直接等刷新/升档；

平时用 /status 查余量最稳。

为什么“限速”？

官方之前也说过，很直接：有人把 Claude Code 挂后台 24/7，还有账号共享/转售的违规，用掉太多公共算力；

而周限额用来抑制极端用法，保证多数人稳定可用。

怎么玩得更久？

  • 有 5 小时滚动窗口：同一段 5 小时里把重活都堆进去，容易一下子把当期窗口的额度耗尽；把重活安排在跨两个窗口（比如上午一段、下午隔 5 小时后再一段），就能少卡瓶颈。

    • 例子：9:00–12:00 跑重任务，下一段尽量放到 ≥14:00 以后（与前一次重载相隔 5 小时左右），这样第二段就落在新窗口里，不会因为前一段把桶灌满而被限。

  • 有 7 天周限额：Opus 4 的周配额更紧（区间更小），把 Opus 留给确实需要它的场景（大改造、复杂推演、跨仓深推），能延长到达“周顶”的时间；日常能用 Sonnet 的尽量用 Sonnet。

  • 强度节奏管理：接近周末时主动降强度（尤其 Pro 用户），可以避免“周三、周四就没量”的窘境；Max 用户周末以 Sonnet 为主，Opus 根据周累计再决定是否上。
