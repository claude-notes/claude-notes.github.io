OpenAI推出更擅长AI代理编码的GPT-5-Codex，与Claude code有何区别？国内怎么使用到Codex呢？

OpenAI近日正式发布 GPT-5-Codex，这是一款基于GPT-5进一步优化的版本，专为代理式编码（agentic coding）量身定制。

在Claude Code 时不时降智的时间段，GPT-5-Codex的发布也是引来了不少开发者的围观。

而这次主要更新了什么呢？codex 与 claude code的区别是怎样的呢？我们将一一进行对比

另外文末教你在国内如何畅玩这两大AI助手？

什么是 Codex？

可能有一些朋友还不清楚codex是什么？

Codex 是 OpenAI 推出的“代理式”编码搭子，既能在你本地仓库里读/改/跑代码，也能把任务丢到云端自己搭环境、跑测试、提 PR，还能在 GitHub 做代码审查。

它既有命令行（CLI），也有 IDE 插件（VS Code / Cursor / Windsurf），还支持 Web/移动端入口，账号用 ChatGPT 的就行。

如果你在之前已经了解过Claude code这个强有力的AI编程助手的话，那你可以理解为Codex是OpenAI出品的”Claude code”。

最近更新了什么？

  • GPT-5-Codex 正式上线：在 GPT-5 基础上专门为“代理式编码”做了强化，既能秒回小活儿，也能长时间独立推进复杂任务。官方实测最长可连续工作 7+ 小时，小任务时还能明显减少消耗。

  • 更强的代码审查：能跑代码与测试去验证差异，优先指出高影响的问题，减少无效评论。
  • CLI/IDE/云端全面升级：
    • 终端 UI 更清晰、待办追踪、工具调用更稳
    • IDE 插件支持在本地与云任务间无缝切换
    • 云端任务自动缓存环境、支持可控的联网与截图回传
入口 典型体验 适合做什么 主要优势 什么时候切到它
Codex CLI 和 Claude Code 类似，直接在终端结对编程 修脚本、小改动、跑单测、看日志、快速诊断 打开即用、干净高效、对命令/日志最友好 想“快进快出”、不想开 IDE 的时候
Codex IDE VS Code / Cursor 等熟悉编辑器里用 多文件联动、代码导航、局部/整体重构 无缝预览本地更改、更懂上下文 需要看 Diff、跨文件思考、边写边测
Codex Cloud 完全托管的远程环境 长时间/重任务（依赖复杂、跨仓库改造） 自动拉依赖、配环境、可长时运行 本地跑不动、要挂长任务、让它“自己干到绿”

参考成绩：官方称在 SWE-bench Verified 全 500 题上更新了统计口径，并展示了相较常规 GPT-5 的提升与更灵活的“思考时间”分配。

各版本能用多少次 Codex？

官方给的是区间与节流窗口，不是“固定次数”。核心规则：每 5 小时一档 + 每周上限。

  • Plus(每月 20 美金) ：
    • 本地任务 每 5 小时约 30–150 条消息，并有周上限；
    • 云任务“阶段性更宽松”。适合一周几次专注编码。
  • Pro(每月 200 美金) ：
    • 本地任务 每 5 小时约 300–1,500 条消息，并有周上限；
    • 云任务“阶段性更宽松”。适合整周多项目推进。
  • Business/Edu/Enterprise：
    • 默认与 Plus 接近；
    • Business 可加购额度，Enterprise/Edu 可走共享信用池。
  • 使用 API Key，按标准 API 费率收费 https://openai.com/zh-Hans-CN/api/pricing/。

以上按官网当前展示为准。

体感经验（非官方数值）：社区普遍反馈 Plus=每周几次 1–2 小时的重度会话，Pro=基本覆盖完整工作日/工作周，但仍受 5 小时窗口与周上限影响。

Codex 与 Claude Code （对比表）

维度 OpenAI Codex Anthropic Claude Code
产品定位 终端/IDE/Web/云一体的代理式编码搭子；含 GitHub 审查 以终端为中心的代理式编码工具，强调大代码库理解与重构
默认模型 GPT-5 / GPT-5-Codex（可选推理强度） Claude Sonnet 4；高阶使用含 Opus 4（取决于订阅）
安装命令 npm i -g @openai/codex npm i -g @anthropic-ai/claude-code
IDE 集成 VS Code / Cursor / Windsurf 官方扩展，支持 Diff 预览与云任务切换 VS Code 官方扩展，支持选择上下文、Diff 视图、tab 感知
本地 vs 云 本地读/改/跑 + 云端托管任务 + PR 审查 本地为主，也支持与 IDE 深度融合；云端能力依订阅与产品形态
安全/审批 沙箱默认禁网；只读/自动/全权限三档审批 终端内人机协作为主，依策略控制命令执行与上下文注入
定价与获取 随 ChatGPT 订阅（Plus/Pro/Business/Edu/Enterprise），也可 API Key 按量 随 Claude 订阅（Pro / Max 5× / Max 20× 等）
使用上限（订阅） Plus：每 5 小时约 30–150 本地消息 + 周上限；Pro：300–1,500 + 周上限；Business/Enterprise 可加购/共享信用池 官方宣布自 8/28 起执行周上限；行业报道与社区称 Pro≈40–80 小时 Sonnet/周，Max 5×/20× 成比例放大，Opus 时数更紧（以官方通知为准）
代表能力 云端长时托管（7h+）、PR 审查、自动环境搭建/截图回传 大型代码库理解、重构与解释、VS Code 深融（选择上下文/快捷键等）
适用人群 想把“查问题→写改→测→提审”打通并托管给代理的团队/个人 偏重在终端与 VS Code 内高效来回迭代的个人开发者与小团队
