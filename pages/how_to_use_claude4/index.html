最强编程模型Claude 4正式发布！！人人皆可编程时代来临，那我们如何体验到最新模型？

就在今天凌晨 00 : 30，Anthropic公司举办了首届开发者大会。

本次大会的主题是：Code with Claude

如果你用过 Cursor 或其他编程工具，你可能会发现，抛开参数和虚幻的排名不谈，Claude 3 系列模型在实际编程体验上最为流畅。

而此次开发者大会也让无数开发者为之欢呼。

Claude 4 今天正式上线！！！

这次 Claude 4 系列总共推出了两个模型：Claude Opus 4 和 Claude Sonnet 4。

Opus以前还是叫 Claude 3 opus，现在数字后置了，改为Claude Opus 4，哈哈~

Claude 4 正式发布

这两个模型专门为编码、高级推理和AI Agent任务而设计。

  • Claude Opus 4：被称为全球最强的编码模型，擅长解决复杂的编程问题，能够自主编程数小时，表现稳定且出色。

  • Claude Sonnet 4：作为Claude Sonnet 3.7的升级版，相较于Opus 4，它更轻便、响应更迅速，适用于实时场景，并且在推理和编程能力上超越其他模型，关键是：免费用户也可使用。

这两个模型都可以在推理过程中使用工具，并且能够在推理和工具使用之间交替进行，甚至可以同时使用多个工具。

常规来看看官方的跑分情况，在行业标准 SWE Bench 数据集上，领先于OpenAI o3、4.1以及Gemini 2.5 Pro等最新发布模型，遥遥领先！！！

稍微解释一下：

SWE-bench Verified: 真实软件工程任务性能基准测试。

各AI模型发布前都会进行的测试，已展示模型的处理任务性能

简而言之，Claude Opus 4 和 Claude Sonnet 4目前是最强的编程模型。

除了编程以外，其他领域也是不相上下

这次的升级不仅让模型变得更聪明（比如学会了用工具、同时处理多个任务、提升记忆力），更重要的是解决了不少以前的漏洞。

更厉害的是，Claude Opus 4的“脑容量”直接爆发！它在内存处理上远超所有以前的版本。当开发者构建允许Claude访问本地文件的应用时，Opus 4能熟练地创建和维护“内存文件”，把重要信息存下来。

针对冗长的思考过程，Claude 4 还引入了思维摘要功能，使用一个较小的模型进行总结。同时，在易受捷径和漏洞影响的智能体任务上，新模型发生这类行为的可能性比 Sonnet 3.7 降低了65%。对于因安全原因拒绝生成内容的情况，Claude Sonnet 4 也是引入了新的 refusal 停止理由。

这样，它的记忆力比之前的版本强了太多。

举个例子，用它玩宝可梦游戏时，它居然会自己写攻略地图！一路走到哪儿，路线标记就实时更新，打过的架、藏着的道具一清二楚。它的记忆力真是超强，处理连续任务时特别靠谱，就像有人帮你把工作日志写得明明白白，复杂的任务也不会出错。

Claude Code

此外，Anthropic还发布了Claude Code， 将 Claude 的强大能力更深入地整合到开发工作流中。Claude Code 现已正式发布，支持通过 GitHub Actions 执行后台任务，并与 VS Code 和 JetBrains 等主流IDE原生集成，实现无缝的结对编程体验。

除了前面提到的，这次它们在API方面也做了不少改进，推出了四项新功能，让开发者能打造更强大的AI助手：代码执行工具、MCP连接器、文件API，以及支持提示缓存最长可达一小时。

这意味着，以后我们的Cursor会更加顺手，这也确实是一个好消息。

现在我在X上看到不少开发者分享自己用Claude 4的经验。有些人只用了30秒就做出了CRM仪表盘，看来Claude 4的效率真是提高了。它不仅速度更快，写的代码更多，关键是理解能力变得更精准，少了很多绕路，整体上效率大大提升。

还有的在用一句话生成一个可用的浏览器代理，也是让老哥爆了回粗口

有的还用2分钟自动开发出俄罗斯方块，赚钱小游戏等等，看着页面也挺吸引人的，速度质量双在线！

  • API方面，模型定价与之前的 Opus 和 Sonnet 模型保持一致：
    • Opus 4 为 每百万输入/输出 tokens 分别为15美元 / 75美元
    • Sonnet 4 为 每百万输入/输出 tokens 分别为3美元 / 15美元
